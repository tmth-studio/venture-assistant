# /session-prep

Generate a pre-session note for Tom — a structured brief Ben sends before each coaching call, summarising what was worked on between sessions.

**When to run:** When Ben has completed a block of between-session work and the next call with Tom is approaching. Also prompt Ben to run this if a natural stopping point has been reached (stage gate passed, a requirement iterated, or a significant stuck point surfaced).

---

## What to Do

1. Read `03-Business_Architecture/Venture_Design_Progress.md` to establish which stages have been completed.
2. Check `07-Coach-Review/` for the most recent session prep file — use its date to identify what's new since the last call.
3. Read the new or updated files from that period (`02-Requirements/`, `03-Business_Architecture/`, `04-Financial_Simulation/`) — focus on what changed, not the full history.
4. Generate the pre-session note (format below).
5. Save to `07-Coach-Review/[YYYY-MM-DD]-session-prep.md`.
6. Tell Ben: "Here's your pre-session note. Copy this and send it to Tom before your call — WhatsApp or email. He reads it in the 5 minutes before you connect."

---

## Pre-Session Note Format

Under 300 words. Tom reads this cold, so write it as if he hasn't seen any of the session work.

```
Pre-session note — [date]

What we worked on:
[Which stages — PCO, R1, R2, etc. — and the core question each was solving. One sentence per stage.]

Key outputs:
- [The actual answer Ben reached, in his words — not a paraphrase. Tom wants the real content, not a summary of it.]
- [If a stage was iterated, name the iteration and what changed.]
- [3 bullets maximum.]

Where I got stuck or forced an answer:
[Any point where Ben felt resistance, doubt, or answered before genuinely listening. Name the specific question and what came up — not just "R2 felt hard."]

What I want Tom to challenge:
[One specific thing for Tom to push on — a theorem that feels shaky, an assumption that might not hold, something that didn't fully resolve. Be precise.]

Proposed agenda item for the call:
[One question or decision.]
```

---

## Notes

- If no progress file exists yet, tell Ben: "There's nothing saved yet — work through at least one stage first, then run `/session-prep`."
- If Ben ran `/session-prep` recently and hasn't done more work since, say so: "The last note was sent [date] — no new work has been logged since then."
- Do not summarise the methodology back to Ben in the note. Tom knows the methodology. Write what Ben produced and where it lives.
